# NL Table — Контекст проекта

## Пользовательские уровни

1. **🟤 Гость** — не зарегистрирован, видит лендинг/демо
2. **🟡 Зарегистрирован** — email + пароль, но нет организаций (WB API ключ не добавлялся)
3. **🟠 Org + ключ невалиден** — история сохранена, синк не работает, предупреждение
4. **🟢 Org + валидный ключ** — всё работает, данные обновляются

## Регистрация и авторизация

- Регистрация: вводит email → система генерит пароль → отправляет на почту
- Логин: email + пароль из письма
- Org_id НЕ создаётся при регистрации, только при добавлении первого WB API ключа
- Из WB API ключа подтягивается информация кабинета → создаётся организация

## Организации

- Одна почта → несколько организаций (несколько кабинетов WB)
- В шапке НЛ — выпадающий список организаций
- Переключение между организациями без перезагрузки

## Данные

- При инвалидации ключа: история и данные пользователя остаются
- reference_sheet (себестоимость) — всегда доступна, это данные пользователя
- raw_api_data — не удаляются при инвалидации ключа
- Не перегружать БД лишними данными

## Справочный лист (вкладка 1)

- Дата-пикер — выбирает дату, видит артикулы/СКЮ/арт WB из ТС на эту дату
- Поля ввода: себестоимость (первое обязательное), закупочная, упаковка, логистика, прочее, заметки
- Ввод вручную или через Excel файл (план)
- Себестоимость привязана к (org_id, nm_id, target_date)

## Стек

- Backend: FastAPI + SQLAlchemy async + PostgreSQL + Celery + Redis
- Frontend: встроенный HTML/JS
- Сборщик: 7 WB API методов, Celery beat раз в 24ч
